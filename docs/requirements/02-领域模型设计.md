# FlowPulse 领域模型设计

## 一、核心领域概念

FlowPulse 的核心领域由以下对象组成：

| 领域对象 | 含义 |
|---|---|
| 环境 | 数据流隔离边界，例如生产环境、测试环境 |
| 区域 | 环境下的管理区或计算区 |
| OMP 配置 | 区域的平台/OMP 接入信息 |
| 基础设施 | Kafka、Spark、Elasticsearch 等基础设施实例 |
| 基础设施资源 | Topic、Index、Spark 作业等可监控对象 |
| 应用服务 | 被监控应用的逻辑服务 |
| 应用实例 | 应用集群中的实例，支持主节点/从节点 |
| 执行节点 | 脚本指标的执行机器，支持 SSH/Agent 能力标识 |
| 数据流 | 一条业务数据链路 |
| 真实关系 | 两个真实资源之间的数据流向关系 |
| 拓扑视图 | 某条数据流的画布表现 |
| 画布节点 | 拓扑视图里的节点图形 |
| 画布连线 | 拓扑视图里的连线图形 |
| 指标定义 | 可采集或可衍生的指标元数据 |
| 指标采集配置 | 某个资源/关系如何采集某个指标 |
| 指标展示配置 | 某个拓扑上展示哪些指标 |
| 阈值配置 | 指标级别判定规则 |
| 告警状态 | 当前资源或关系的告警级别 |
| 告警历史 | 告警级别变化记录 |
| 故障事件 | 一次连续异常过程的聚合 |

## 二、环境与区域

### 2.1 环境

环境是数据流的强隔离边界。

规则：

1. 数据流不允许跨环境。
2. 同一环境下可以跨区域建数据流。
3. 环境用于资源、区域、基础设施、应用和数据流的归属划分。

### 2.2 区域

区域分为：

| 区域类型 | 说明 |
|---|---|
| 区域 | 承载平台访问地址、OMP 地址、平台 API Key、OMP API Key 等配置 |
| 计算区 | 承载资源和业务运行环境，必须归属于一个管理区 |

规则：

1. 计算区必须归属于同环境下的管理区。
2. 同一个区域最多配置一组平台与 OMP 接入信息。
3. OMP 配置边界、SSH 执行节点配置边界归入环境区域页面说明。

## 三、基础设施与资源

基础设施是可连接、可同步资源、可采集自身指标的外部系统实例。

一期基础设施类型：

1. Kafka。
2. Spark。
3. Elasticsearch。

基础设施资源是基础设施下可被拓扑选择的对象：

| 基础设施 | 资源示例 |
|---|---|
| Kafka | Topic、Consumer Group |
| Spark | Spark 作业、应用实例 |
| Elasticsearch | Index、Alias |

应用服务不属于某个基础设施，它属于应用监控领域。拓扑中可以选择应用节点，但应用监控单独设计页面。

## 四、真实对象与画布对象

### 4.1 真实对象

真实对象包括：

1. Kafka Topic。
2. Kafka Consumer Group。
3. Spark 作业。
4. ES Index。
5. 应用服务。
6. 应用实例。
7. 其他后续扩展资源。

真实对象必须有稳定 ID，不能使用画布随机 ID。

### 4.2 真实关系

真实关系表示两个真实对象之间存在数据流向，例如：

1. Kafka Topic -> Spark 作业。
2. Spark 作业 -> ES Index。
3. 应用服务 -> Kafka Topic。

真实关系也必须有稳定 ID。

关系 ID 不应该依赖画布线条 ID，而应该由真实源对象、目标对象、关系类型和必要业务参数确定。

### 4.3 拓扑视图

拓扑视图是数据流在画布上的表达。

画布节点保存：

1. 位置。
2. 尺寸。
3. 标签位置。
4. 展示样式。
5. 当前拓扑指标展示配置。
6. 是否隐藏或分组。

画布连线保存：

1. 路径。
2. 锚点。
3. 标签位置。
4. 展示样式。
5. 当前拓扑指标展示配置。
6. 是否隐藏。

画布不保存：

1. 真实对象身份。
2. 真实关系身份。
3. 监控配置。
4. 阈值。
5. 告警。

画布节点/连线通过引用真实对象 ID/真实关系 ID 进行关联。

## 五、节点问题与连线问题

节点和连线都可以有指标，也都可以产生告警。

| 告警位置 | 示例 |
|---|---|
| 节点告警 | Spark 作业停止、ES Index 文档数不增长 |
| 连线告警 | Topic 到 Spark 的消费延迟过高、Spark 到 ES 写入速率为 0 |

用户在拓扑上看到异常时，需要能明确：

1. 是节点自身异常。
2. 是连线关系异常。
3. 是上游异常导致当前节点无数据。

不再使用“影响状态”作为节点独立状态字段。上游影响可以通过告警描述、关联原因、链路诊断说明表达，避免把上游 Spark 停止强行显示成 ES 节点自身严重。

## 六、告警故障

故障事件不是页面概念，而是告警领域模型。

一次故障事件用于聚合同一个对象、同一个指标、同一轮连续异常过程中的多次告警级别变化。

规则：

1. 初始状态为正常。
2. 只要告警级别发生变化，就记录历史并发送通知。
3. 从正常变为非正常时创建故障事件。
4. 非正常级别之间变化时归入同一故障事件。
5. 从非正常恢复正常时关闭故障事件。
6. 恢复通知也必须发送。
