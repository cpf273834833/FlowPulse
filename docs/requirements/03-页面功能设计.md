# FlowPulse 页面功能设计

## 一、页面导航

一期页面建议按下面结构组织：

```text
观测
├── 运行总览
├── 数据流列表
├── 数据流拓扑
├── 告警中心
└── 真实关系

对象
├── 环境区域
├── 基础设施
├── 执行节点
├── 应用监控
└── 数据平台资源

策略
├── 指标中心
├── 阈值管理
├── 通知配置
└── 数据保留
```

权限一期只要求能进行菜单和按钮控制，具体接入公司平台权限方案后续按脚手架调整。

## 二、统一页面设计原则

1. 每个主页面保留统计区，但统计区不能过高，不能压缩主要内容。
2. 标题需要醒目，标题说明不要和页面内规则卡片重复。
3. 二级页面是真正的页面，不使用弹窗替代详情页。
4. 一级页面不需要面包屑；二级页面使用面包屑返回。
5. 新增、编辑、删除等操作需要明确确认和反馈。
6. 页面需要自适应，拓扑页面尽可能给画布留空间。
7. 风格参考公司脚手架，但不能被脚手架限制，视觉需要更现代、更清晰。

## 三、环境区域页面

### 3.1 页面目标

维护环境、管理区、计算区以及各区域的平台/OMP 配置。

### 3.2 页面结构

一级页面展示：

1. 统计区。
2. 环境与区域关系。
3. 每个环境下有哪些管理区。
4. 每个管理区下有哪些计算区。
5. 每个区域的关键配置状态。
6. 新增环境入口。

去掉：

1. 独立“区域和平台配置”菜单。
2. 二级维护页面。
3. 重复规则说明卡片。
4. 与统计区重复的“当前配置状态”卡片。

### 3.3 操作

| 操作 | 说明 |
|---|---|
| 新增环境 | 点击“新增环境”弹出环境信息表单 |
| 新增管理区 | 在环境下新增 |
| 新增计算区 | 在管理区下新增 |
| 修改环境 | 页面直接修改 |
| 修改区域 | 统一入口，区域信息、平台配置、OMP 配置都在修改中完成 |
| 删除 | 删除前必须确认 |

### 3.4 关键规则

1. 计算区必须归属于管理区。
2. 管理区归属于环境。
3. 数据流允许跨区域，不允许跨环境。
4. 同一管理区只能有一个 OMP。
5. OMP 应属于环境区域配置信息，不属于基础设施。

## 四、基础设施页面

### 4.1 页面目标

维护 Kafka、Spark、Elasticsearch 等基础设施连接配置，作为资源发现、资源同步和基础设施自身监控的入口。

### 4.2 主页面

主页面包含：

1. 统计区。
2. 基础设施列表标题。
3. 筛选区，筛选区属于基础设施列表。
4. 卡片列表。

卡片设计要求：

1. 卡片不能过大，一屏需要能看到多个实例。
2. 实例边界清晰，避免密集拥挤。
3. 点击名称进入详情，卡片其他区域不进入详情。
4. 操作按钮不能视觉过重。
5. 启停使用开关按钮，带左右滑动效果和文字。
6. 启用必须先测试通过。
7. 停用后如果连接状态是正常，改为未测试；异常不改。
8. 连接状态使用图标和颜色，不使用大段汉字标签。
9. 实例类型作为醒目标签展示。
10. 名称需要限制长度。
11. 元信息一行展示，值右对齐，时间字号略小。
12. 页面提示使用小窗口提示，确认后消失，不放在统计区下面。

### 4.3 详情页

详情页只展示详情，不显示主页统计。

详情页包含：

1. 面包屑返回基础设施列表。
2. 标题区。
3. 启用状态和连接状态以图标、颜色、文字组合表达，不重复放入基本信息。
4. 描述放在右侧操作按钮下方，并和左侧状态区域对齐。
5. 基本信息。
6. 连接配置。
7. 资源同步配置。
8. 已同步资源分页列表，支持搜索。
9. 运行记录。
10. 操作记录。

卡片上的操作按钮去掉，操作集中在详情页。

### 4.4 测试连接

测试连接需要：

1. 有加载动画。
2. 最后显示成功或失败。
3. 卡片上能看到最近测试结果。
4. 测试结果 3 秒后自动消失。

### 4.5 资源同步

统一使用“同步资源”命名，不再一处叫刷新资源、一处叫同步资源。

同步后的资源保存到数据库。

资源同步范围由基础设施配置决定，例如：

| 类型 | 同步范围示例 |
|---|---|
| Kafka | Topic 名称通配或正则，例如 `dw_*` |
| ES | Index/Alias 通配或正则 |
| Spark | 作业名称、应用名称或队列范围 |

同步范围变更后，需要考虑已有同步资源：

1. 新范围内发现的资源更新或新增。
2. 不在新范围内的旧资源标记为失效或不可见，不能直接物理删除，避免影响历史告警和拓扑引用。

## 五、执行节点页面

### 5.1 页面目标

维护脚本指标采集的执行机器。

执行节点不是服务端节点，也不是被监控应用节点。它表示“脚本可以在哪台机器上通过什么方式执行”。

### 5.2 主页面

执行节点主页面不使用卡片，使用列表分页展示，因为节点可能很多。

列表字段：

1. 主机地址。
2. 环境。
3. 区域。
4. 来源：OMP 或人工新增。
5. SSH 是否可用。
6. Agent 是否可用。
7. 连接状态。
8. 最近测试时间。
9. 启用状态。
10. 操作。

点击 IP 进入详情页面。

### 5.3 详情页

详情页包含：

1. 面包屑返回。
2. 主机地址标题。
3. 基本信息。
4. SSH 配置。
5. Agent 状态预留。
6. 连接测试。
7. 运行记录。
8. 操作记录。

测试连接功能在详情页。

点击修改按钮进入修改页面字段配置。

### 5.4 规则

1. IP 不允许修改；如果 IP 不对就删除后重新新增。
2. 同一环境、同一区域下不允许重复 IP。
3. 不需要节点编码、节点名称，主机地址就是标识。
4. 搜索以主机地址为主。
5. 节点描述字段去掉。
6. SSH 配置正常则支持 SSH。
7. Agent 后续通过心跳判断是否支持。
8. 没有首选通道概念；脚本执行方式由脚本采集配置决定。
9. 支持从各区域 OMP 同步执行节点。
10. 从 OMP 同步必须按指定区域同步，即使用该计算区所属管理区的 OMP 配置。

## 六、拓扑页面

拓扑页面核心是完整数据流拓扑图。

要求：

1. 左侧筛选可折叠，尽量给画布留空间。
2. 右侧详情栏标题不能被圆角裁切。
3. 节点指标展示紧凑，避免指标上下堆叠导致节点过高。
4. 拓扑指标展示配置是每个拓扑自己的配置。
5. 拓扑页面可以进入阈值配置，但阈值本身不是画布数据。
6. 可以有轻微 3D 视觉效果，但一期优先 2D 清晰表达。
