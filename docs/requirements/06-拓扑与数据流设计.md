# FlowPulse 拓扑与数据流设计

## 一、拓扑目标

拓扑图是 FlowPulse 的核心页面。

用户通过拓扑图需要看到：

1. 数据从哪个 Kafka Topic 来。
2. 被哪个 Spark 作业或应用处理。
3. 写入哪个 ES Index 或其他目标。
4. 哪个节点异常。
5. 哪条关系异常。
6. 异常指标是什么。
7. 告警级别如何。
8. 最近数据更新时间和关键指标。

## 二、一期建图方式

一期拓扑完全人工创建。

规则：

1. 数据流由用户创建。
2. 节点只能从已有资源中选择。
3. 不允许建图时顺便创建资源。
4. 资源创建和同步在独立页面完成。
5. 允许跨区域建数据流。
6. 不允许跨环境建数据流。

## 三、数据流与拓扑关系

数据流是业务链路。

拓扑视图是数据流的一种画布呈现。

真实关系是数据流中的真实数据流向。

三者关系：

```text
数据流
├── 真实关系 1
├── 真实关系 2
└── 拓扑视图
    ├── 画布节点
    └── 画布连线
```

## 四、真实关系复用

如果一个拓扑关系在另一个拓扑中已经存在：

1. 新拓扑添加两个节点时，不自动强制出现关系。
2. 创建连线时系统需要检查真实关系是否已存在。
3. 如果已存在，提示用户复用已有真实关系。
4. 如果不存在，创建新的真实关系。
5. 同一个真实关系可以出现在多个拓扑中。

原因：

1. 两个资源之间可能存在多种关系。
2. 用户未必希望所有关系都展示在当前拓扑。
3. 拓扑是视图，不是全局关系自动渲染器。

## 五、节点设计

拓扑节点引用真实资源。

节点展示：

1. 资源名称。
2. 资源类型。
3. 当前告警状态。
4. 当前拓扑配置的展示指标。
5. 最新更新时间。

节点指标：

1. 节点可以展示多个指标。
2. 指标展示要紧凑，避免节点过高。
3. 哪些指标展示由当前拓扑单独配置。

## 六、连线设计

拓扑连线引用真实关系。

连线展示：

1. 数据方向。
2. 关系类型。
3. 当前告警状态。
4. 当前拓扑配置的展示指标。

连线 ID：

1. 画布连线 ID 只用于画布元素。
2. 真实关系 ID 用于监控、阈值、告警。
3. 画布连线通过 realRelationId 引用真实关系。

## 七、节点指标与连线指标

节点和连线都可以配置指标。

节点指标示例：

1. Spark 作业是否运行。
2. ES Index 文档数。
3. Kafka Topic 分区数。

连线指标示例：

1. Kafka Topic 到 Spark Consumer Group 的 Lag。
2. Spark 到 ES 的写入速率。
3. 最后数据更新时间。

## 八、最大数据未更新时间

“最大数据未更新时间”本质是一个指标。

如果节点或连线需要展示该能力，应通过指标定义新增对应指标，而不是做成拓扑硬编码字段。

## 九、应用主节点消费 Topic

一期应用实例角色：

1. 主节点。
2. 从节点。

如果只有主节点消费某个 Kafka Topic：

1. 应用服务节点需要知道当前主节点是谁。
2. 主节点切换后，关系指标应动态使用新的主节点。
3. 该能力需要在应用监控设计中支持节点角色。

链路是否正常不能只靠一个指标，需要衍生指标组合：

1. 主节点是否在线。
2. 当前主节点是否匹配。
3. Topic 是否有消息。
4. Consumer Group 是否活跃。
5. Lag 是否异常。
6. Offset 是否更新。

## 十、拓扑指标展示配置

拓扑展示哪些指标是每个拓扑自己的配置。

原因：

1. 同一个真实资源出现在不同拓扑中，关注指标可能不同。
2. 展示配置属于视图。
3. 阈值配置属于监控规则。

## 十一、拓扑内阈值配置

拓扑页面必须支持阈值配置入口。

流程：

1. 用户在节点或连线上点击配置阈值。
2. 系统根据真实资源或真实关系查询当前指标阈值。
3. 查询该阈值被哪些拓扑使用。
4. 未被使用则直接保存。
5. 已被其他拓扑使用则提示影响范围。
6. 用户确认后保存。

## 十二、拓扑视觉

一期以清晰 2D 拓扑为主。

可以做轻微 3D 效果，例如：

1. 节点阴影。
2. 层级光效。
3. 轻微透视背景。

但不能影响：

1. 节点密度。
2. 指标可读性。
3. 告警状态识别。
4. 页面性能。
