# FlowPulse 基础设施与执行节点设计

## 一、基础设施定位

基础设施是外部系统接入 FlowPulse 的入口，负责：

1. 保存连接配置。
2. 测试连接。
3. 发现和同步资源。
4. 提供基础设施自身监控指标。

一期基础设施类型：

1. Kafka。
2. Spark。
3. Elasticsearch。

应用服务不归属于基础设施。应用监控单独设计页面，但拓扑可以选择应用节点。

## 二、基础设施字段

### 2.1 通用字段

| 字段 | 说明 |
|---|---|
| id | 32 位字符串主键 |
| tenantId | 32 位租户 ID |
| envId | 所属环境 |
| regionId | 所属区域 |
| infraCode | 基础设施编码 |
| infraName | 基础设施名称 |
| infraType | KAFKA / SPARK / ELASTICSEARCH |
| primaryEndpoint | 连接地址 |
| status | ENABLED / DISABLED |
| connectionStatus | UNTESTED / NORMAL / ERROR / UNKNOWN |
| syncEnabled | 是否启用资源同步 |
| syncIntervalSec | 同步周期 |
| lastTestAt | 最近测试时间，毫秒时间戳 |
| lastSyncAt | 最近同步时间，毫秒时间戳 |
| owner | 负责人 |
| tags | 标签 |
| description | 描述 |
| createdAt | 创建时间，毫秒时间戳 |
| updatedAt | 更新时间，毫秒时间戳 |

### 2.2 连接与认证字段

| 字段 | 说明 |
|---|---|
| authType | NONE / BASIC / TOKEN / API_KEY 等 |
| username | 用户名 |
| password | 密码，后续需要加密存储 |
| token | Token 或 API Key，后续需要加密存储 |
| connectTimeoutSec | 连接超时 |
| requestTimeoutSec | 请求超时 |
| verifyCert | 是否校验证书 |

### 2.3 类型扩展字段

不同引擎连接参数不同，设计上必须支持类型差异。

一期先使用明确字段 + 扩展参数：

| 类型 | 关键字段 |
|---|---|
| Kafka | bootstrap servers、securityProtocol、认证配置 |
| Spark | master/history/server URL、sparkAccessMode |
| Elasticsearch | http URL、认证配置、证书校验 |

后续如果字段变复杂，可以演进为：

1. 类型定义表。
2. 参数 schema。
3. 动态表单。
4. 参数加密策略。

## 三、Kafka 连接设计

Kafka 多地址配置优先交给官方客户端机制。

说明：

1. 用户填写多个 bootstrap servers。
2. FlowPulse 不逐个手写尝试。
3. Kafka AdminClient 自己会选择可用 broker 并处理元数据发现。
4. FlowPulse 只负责设置合理超时和认证参数。

测试连接：

1. 创建 AdminClient。
2. 拉取集群元数据或 topic 列表。
3. 成功则连接正常。
4. 失败记录错误原因。

资源同步：

1. 按同步范围发现 Topic。
2. 可同步 Consumer Group。
3. Consumer Group 归属关系由关系配置或消费组选择器确定。

## 四、Elasticsearch 连接设计

ES 多地址配置优先交给 HTTP 客户端或 ES 官方客户端。

测试连接：

1. 请求 `_cluster/health` 或根路径。
2. 根据认证配置设置 Basic/Auth Token。
3. 401 表示认证失败。
4. 连接成功不代表指标权限成功，指标能力需分项检测。

资源同步：

1. 按同步范围发现 Index/Alias。
2. 已不在范围内的资源标记失效，不直接删除。

## 五、Spark 连接设计

Spark 一期支持 Standalone Master / History Server / REST 地址。

连接测试：

1. 请求 Spark Web UI 或 API。
2. 对返回 JSON 做容错处理，不能因为响应过长或截断导致整体崩溃。
3. 连接测试和资源同步需要明确错误原因。

资源同步：

1. 同步应用或作业。
2. 记录作业状态、最近看到时间。
3. 后续可关联 Kafka 消费组规则。

## 六、资源同步设计

同步后的资源必须保存数据库。

资源生命周期：

| 状态 | 说明 |
|---|---|
| ACTIVE | 最近同步中仍存在 |
| LOST | 曾经存在，但当前同步未发现 |
| DISABLED | 人工停用 |

同步策略：

1. 新发现资源：新增。
2. 已存在资源：更新属性、lastSeenAt。
3. 不在当前结果中但历史存在：标记 LOST。
4. 被拓扑引用的资源不能物理删除。

## 七、执行节点定位

执行节点是脚本指标采集的执行机器。

它不代表：

1. FlowPulse 服务端。
2. 被监控应用。
3. 基础设施实例。

它代表：

1. 这台机器可以通过 SSH 执行脚本。
2. 后续 Agent 正常心跳时可以通过 Agent 执行脚本。

脚本如何执行由脚本采集配置决定，而不是执行节点自己决定。

## 八、执行节点字段

| 字段 | 说明 |
|---|---|
| id | 32 位字符串 |
| tenantId | 32 位租户 ID |
| envId | 所属环境 |
| regionId | 所属区域 |
| host | 主机地址，作为业务标识 |
| sourceType | MANUAL / OMP |
| sshEnabled | SSH 是否配置可用 |
| agentEnabled | Agent 是否可用 |
| sshHost | SSH 主机地址 |
| sshPort | SSH 端口 |
| sshUsername | SSH 用户 |
| sshAuthType | PASSWORD / PRIVATE_KEY |
| sshPassword | SSH 密码，需加密存储 |
| privateKey | 私钥，需加密存储 |
| workDir | 工作目录 |
| charset | 字符集 |
| connectionStatus | UNTESTED / NORMAL / ERROR |
| shellStatus | Shell 能力状态 |
| pythonStatus | Python 能力状态 |
| lastTestAt | 最近测试时间 |
| status | ENABLED / DISABLED |

## 九、执行节点规则

1. 主机地址不允许修改。
2. 同一环境、同一区域下主机地址唯一。
3. 不需要节点编码和节点名称。
4. 不需要描述字段。
5. 搜索以主机地址为主。
6. 测试连接暂时以 SSH 连通性为准。
7. SSH 配置通过则支持 SSH 执行。
8. Agent 后续由心跳状态判断是否支持。
9. 从 OMP 同步需要指定区域。
10. 指定计算区时使用其所属管理区的 OMP 配置。

## 十、OMP 同步执行节点

同步入口：

```text
POST /frontapi/v1/execution-nodes/sync-omp
```

请求：

```json
{
  "regionId": "计算区ID"
}
```

处理：

1. 查询 regionId 对应区域。
2. 如果是计算区，找到父管理区。
3. 查询管理区 OMP 配置。
4. 调用 OMP 主机查询接口。
5. 将返回主机写入执行节点表。
6. 已存在主机更新来源、状态和同步时间。
7. 不覆盖用户手工维护的 SSH 密码、私钥等敏感配置。
