# FlowPulse 迭代计划与验收标准

## 一、当前优先级

当前优先完成三个页面：

1. 环境区域。
2. 基础设施。
3. 执行节点。

这三个页面是后续拓扑、指标、告警的基础。

## 二、第一阶段目标

第一阶段完成企业级基础管理能力。

必须满足：

1. 后端使用真实 H2 文件数据库。
2. 使用 MyBatis。
3. 使用真实业务表。
4. 不使用内存数据。
5. 不使用假数据。
6. 不使用万能 JSON Store。
7. Controller、Service、Repository、Mapper、DTO、Entity 分层清晰。
8. 前端页面风格统一。
9. 页面功能可真实操作。
10. 数据刷新后仍然存在。

## 三、环境区域验收

| 验收项 | 标准 |
|---|---|
| 新增环境 | 可新增，刷新后存在 |
| 修改环境 | 可修改，字段清晰 |
| 删除环境 | 有确认；存在区域时不允许删除 |
| 新增管理区 | 可在环境下新增 |
| 新增计算区 | 必须选择同环境管理区 |
| 修改区域 | 一个入口完成区域信息和平台配置 |
| 删除区域 | 有确认；管理区下有计算区时不允许删除 |
| OMP 配置 | 同一区域最多一组 |
| 页面关系 | 首屏能看到环境、管理区、计算区归属关系 |

## 四、基础设施验收

| 验收项 | 标准 |
|---|---|
| 新增基础设施 | 数据保存数据库 |
| 修改基础设施 | 敏感字段不被浏览器自动填充覆盖 |
| 删除基础设施 | 有确认；被引用时后续需限制 |
| 启用 | 必须测试通过后才能启用 |
| 停用 | 正常连接状态改未测试，异常不改 |
| 测试连接 | 有加载态、成功/失败态、错误原因 |
| 同步资源 | 同步结果入库，资源列表可分页搜索 |
| 详情页 | 真正二级页面，不展示主页统计 |
| 卡片 | 紧凑、边界清晰、只点击名称进详情 |
| 排序 | 按创建时间倒序 |

## 五、执行节点验收

| 验收项 | 标准 |
|---|---|
| 列表 | 分页表格展示，不使用卡片 |
| 搜索 | 支持按主机地址搜索 |
| 新增 | 主机地址必填 |
| 唯一性 | 同环境同区域 IP 不允许重复 |
| 修改 | IP 不允许修改 |
| 删除 | 有确认 |
| 详情 | 点击 IP 进入 |
| SSH 测试 | 在详情页执行，显示结果 |
| OMP 同步 | 按指定区域同步，使用管理区 OMP 配置 |
| 来源 | 标识 OMP 或人工新增 |

## 六、第二阶段目标

第二阶段完成拓扑基础。

1. 数据流列表。
2. 资源选择器。
3. 真实关系管理。
4. 拓扑画布。
5. 画布节点/连线保存。
6. 节点/连线指标展示配置。
7. 拓扑内阈值配置入口。

## 七、第三阶段目标

第三阶段完成指标与告警。

1. 指标定义。
2. 指标采集配置。
3. Shell/Python 脚本指标。
4. 衍生指标。
5. 阈值管理。
6. 告警状态。
7. 告警历史。
8. 故障事件。
9. 第三方通知平台对接。

## 八、质量验收标准

后端：

1. 编译通过。
2. 单元测试覆盖核心规则。
3. 平台 API Key 与 OMP API Key 按当前产品规则允许明文保存和回显。
4. 数据库约束能防止脏数据。
5. 删除、更新有业务校验。
6. 异常统一格式。
7. 日志不输出密码、私钥等敏感信息；API Key 按当前产品规则不加密。

前端：

1. 页面风格统一。
2. 组件复用。
3. 表单字段与字段值区分清晰。
4. 下拉框、按钮、提示风格统一。
5. 页面自适应。
6. 错误提示可见。
7. 操作反馈明确。

文档：

1. 每个功能模块有中文需求文档。
2. 领域规则写清楚。
3. 数据库字段写清楚。
4. 页面交互写清楚。
5. 后续开发按文档同步更新。
